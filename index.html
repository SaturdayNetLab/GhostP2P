Security Architecture

ECDH P-256 Key Exchange

Ephemeral keypairs generated per session. Every connection has unique shared secrets. Forward secrecy guaranteed.

AES-256-GCM Encryption

Application-layer encryption on top of WebRTC DTLS. Messages, files, images, and voice are all encrypted before transmission.

PBKDF2 Password Hashing

Room passwords are derived using 100,000 iterations with random salt. Passwords never leave your device in plaintext.

RAM Only — Zero Persistence

Nothing is written to disk. Close the tab and all data — keys, messages, files — is wiped from memory instantly.

XSS Protection

All user-generated content is sanitized through DOMPurify before rendering. No code injection possible.

Serverless P2P

A public STUN server is used only for the initial handshake. After that, all data flows directly between peers. No middleman.

How to Use

1
Create or Join a Room

Click Host Room to create a new encrypted room. Optionally set a password or self-destruct timer. Or paste a Room ID and click Connect to join an existing room.

2
Share the Room ID

Click the Room ID in the header to copy it, use the button for a full invite link, or the button for a QR code.

3
Chat, Send Files & Voice

Type messages with Markdown support. Drag & drop files or paste screenshots with Ctrl+V. Click the button to record and send encrypted voice messages.
